docker安装

emmmm 我对于这个也不是很了解 大家网上搜索一下吧，有很多教程
我搭建时候使用的是云机Ubuntu 18.4系统

然后就是为什么我用了云机呢，因为他几乎不存在镜像拉取失败的问题，比较的方便

github下载开源项目

命令：git clone https://github.com/zhl2008/awd-platform.git
这里文件还是蛮大的，需要等待一段时间
下载好后 cd awd-platform-master进入文件夹中

拉取docker镜像

方法一

如果不是使用云机搭建或docker没有加速建议使用方法二
命令：docker pull zhl2008/web_14.04
然后这里需要等待较长的一段时间，镜像有1G大小

拉取完镜像后修改一下镜像名字： docker tag zhl2008/web_14.04 web_14.04

然后如果说已经做到了这里，没有报错，那么就可以运行了
启动环境：root权限运行，在awd-platform-master目录下

1、
创建队伍数量及靶机 web_server是靶机、2是靶机数量也是队伍数量
python batch.py web_yunnan_simple 2

2、
运行start.py脚本，2是靶机数量，与第一步创建时相对应即可
python start.py ./ 2

3、
启动裁判靶机（check脚本）
先：docker attach check_server
再：python check.py

4、
关闭所有靶机
python stop_clean.py

方法二

本不应该存在方法二的，但是因为docker没有加速真的拉取不来镜像
于是我将镜像打包出来了，所以使用的时候将镜像从本地导入一次就可以使用了
链接：https://pan.baidu.com/s/1xkm7B9PNv5Ilu0v1vzJi9A
提取码：5fck

docker镜像导入命令
二选其一应该都可以，我用了第二种
sudo docker load <　/home/ubuntu/web_14.04.tar
sudo docker load -i /home/ubuntu/web_14.04.tar
这里后面的路径改成对应自己文件的路径就可以了


docker镜像打包命令
sudo docker save -o quay.io-calico-node-1.tar quay.io/calico/node

镜像导入后，docker images查看一下镜像名字是否为 web_14.04
如果不是执行：docker tag zhl2008/web_14.04 web_14.04

然后就可以按照方法一的结尾去启动环境了

但是作为一个日常训练学习的平台还是足够的

这里存在的主要bug出现在flag_server/flag_file.php文件中
原版的代码被我修改了找不到了，记得原版好象是基于时间判断flag提交
原版存在的问题是：

1、flag提交时间间隔必须是2分钟(自定)，即便flag刷新，间隔时间也不会刷新

为了解决上诉问题，我采用了读取flag文件md5的方式，判断两次提交时文件内容是否发生变化

function update_time($attack_uid,$victim_uid){
    global $time_file;
    global $now_time;
    global $team_number;
    global $time_file;
    $old_times = explode('|' , file_get_contents($time_file));        //读取time.txt文件的时间————>old_times
    $id = $attack_uid * $team_number + $victim_uid ;                    //加分*队伍数+减分————>id + $victim_uid
    $old_times[$id] = readmd5();                                    //  
    $now_times = $old_times;                                        //  old_times————>now_time
    file_put_contents($time_file,implode('|' , $now_times));        //将新时间写入文件time.txt

}
